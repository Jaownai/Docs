DataServiceServer

Server-side API for persistent player data, cross-server messaging, and leaderboards. Wraps ProfileStore with reactive signals, automatic client replication, and built-in OrderedDataStore sync.

🖥  Server  ·  require(DataService).server
Player Management
loadPlayer
yields
DataService:loadPlayer(player: Player) → void

Loads and session-locks a player's profile from the DataStore. Call inside Players.PlayerAdded. Yields until the profile is ready — all other DataService calls assume this has completed.

Note: In useMock mode the call resolves instantly with a blank data table.
ParameterTypeDescription
playerPlayerThe Roblox Player instance to load.
Lua 
-- ServerScriptService
local DataService = require(game.ReplicatedStorage.DataService).server

Players.PlayerAdded:Connect(function(player)
    DataService:loadPlayer(player)
end)

Players.PlayerRemoving:Connect(function(player)
    DataService:removePlayer(player)
end)
removePlayer
DataService:removePlayer(player: Player) → void

Saves the player's profile and releases the session lock. Call inside Players.PlayerRemoving.

Caution: ProfileStore handles cleanup on server shutdown automatically, but always call this explicitly on every remove.
ParameterTypeDescription
playerPlayerThe Player instance to save and release.
Data Operations
get
DataService:get(player: Player, path?: Path) → any

Returns the player's in-memory data. Omit path for the full data table. path accepts a string key or an array of strings for nested access.

ParameterTypeDescription
playerPlayerThe target player.
path optionalstring | {string}Key or nested path. Omit for the full data table.
Returns any
Lua 
-- Full snapshot
local data = DataService:get(player)

-- Single field
local coins = DataService:get(player, "Coins")

-- Nested path
local wins = DataService:get(player, {"Stats", "Wins"})
set
DataService:set(player: Player, path: Path, value: any, dontReplicate?: boolean) → void

Writes a value into the player's data at the given path, replicates the change to the client, and automatically updates any leaderboard whose tracked path matches.

Auto-sync: If path is registered with trackLeaderboard and value is a number, the OrderedDataStore is updated in the background.
ParameterTypeDescription
playerPlayerThe target player.
pathstring | {string}Key or nested path to set.
valueanyValue to store. Must be JSON-serialisable.
dontReplicate optionalbooleanSkip client replication when true.
Lua 
DataService:set(player, "Coins", 500)

-- Nested path
DataService:set(player, {"Stats", "Wins"}, 10)

-- Server-only write (no replication)
DataService:set(player, "SecretFlag", true, true)
update
DataService:update(player: Player, path: Path, fn: (current: any) → any) → any

Reads the current value at path, passes it to fn, then calls set with the returned value. Triggers the same replication and leaderboard sync as set.

ParameterTypeDescription
playerPlayerThe target player.
pathstring | {string}Path to update.
fn(any) → anyReceives the current value; must return the new value.
Returns any — the new value
Lua 
local newCoins = DataService:update(player, "Coins", function(current)
    return (current or 0) + 100
end)

Adds amount (default 1) to the number at path. Shorthand for update with addition. Use a negative value to subtract.

ParameterTypeDescription

Returns number — the new value
Lua

DataService:increment(player, "Level")       -- +1
DataService:increment(player, "Coins", -25) -- -25

asyncGetProfile
yields
DataService:asyncGetProfile(userId: number) → table?

Reads a player's data directly from the DataStore, even if they are not online. Returns nil if no saved data exists. Used internally by getTop to populate extraFields.

Warning: Makes a live DataStore request each call. Cache results where possible.
ParameterTypeDescription
userIdnumberUserId of the player to look up.
Returns table? — raw data table, or nil
Lua 
local profile = DataService:asyncGetProfile(12345678)
if profile then
    print(profile.Coins, profile.Level)
end
Leaderboard new
trackLeaderboard
new
DataService:trackLeaderboard(key: string, path: Path) → void

Binds a leaderboard key to a data path. Every subsequent set or update that writes a number to that path automatically updates the corresponding OrderedDataStore — no extra call required.

Multiple leaderboards can track different paths simultaneously. Call once during server init, before players join.

ParameterTypeDescription
keystringUnique leaderboard name (also the OrderedDataStore name).
pathstring | {string}Data path whose numeric value drives the leaderboard.
getTop
newyields
DataService:getTop(key: string, count: number, extraFields?: {string}) → {LeaderboardEntry}

Returns the top count entries sorted descending by score. When extraFields is provided, each entry is enriched with those data keys fetched concurrently via asyncGetProfile.

ParameterTypeDescription
keystringThe leaderboard key.
countnumberMax entries to return (up to 100 per OrderedDataStore page).
extraFields optional{string}Extra data keys to attach to each entry, e.g. {"Name", "Level"}.
Returns {LeaderboardEntry}
Entry fieldTypeDescription
ranknumberPosition (1 = highest score).
userIdnumberPlayer's UserId.
scorenumberThe tracked value.
...extraFieldsanyAny additional keys you requested.
Lua — trackLeaderboard + getTop 
-- Server init
DataService:trackLeaderboard("richest", "Coins")
DataService:trackLeaderboard("topLevel", "Level")

-- set() now auto-updates OrderedDataStore
DataService:set(player, "Coins", 1500)

-- Fetch top 10, include Name and Level from each profile
local top = DataService:getTop("richest", 10, {"Name", "Level"})
for _, entry in ipairs(top) do
    print(entry.rank, entry.Name, entry.score, entry.Level)
end

Registers a handler for incoming bridge messages. player is non-nil only when the sender used bridgeSendToPlayer and that player is online in this server.

ParameterTypeDescription

callbackfunctionCalled with (data, player?) when a message arrives.
bridgeSend
new
DataService:bridgeSend(key: string, data: any) → void

Broadcasts a message to all game servers (including the sender) via MessagingService.

ParameterTypeDescription
keystringMessage channel name.
dataanyPayload — must be JSON-serialisable.
Lua — bridgeOn + bridgeSend 
-- Listen
DataService:bridgeOn("announcement", function(data, player)
    print("Received:", data.message)
end)

-- Broadcast to all servers
DataService:bridgeSend("announcement", { message = "Event started!" })
bridgeSendToPlayer
new
DataService:bridgeSendToPlayer(userId: number, key: string, data: any) → void

Broadcasts a message only handled by the server where the target player is currently playing. Other servers silently ignore it.

ParameterTypeDescription
userIdnumberUserId of the target player.
keystringMessage channel name.
dataanyPayload — must be JSON-serialisable.
bridgeSendToServer
new
DataService:bridgeSendToServer(jobId: string, key: string, data: any) → void

Sends a message to a specific server by its game.JobId. All other servers silently ignore the message.

ParameterTypeDescription
jobIdstringThe game.JobId of the target server.
keystringMessage channel name.
dataanyPayload — must be JSON-serialisable.
Lua — targeted bridge 
-- Send to a specific server by JobId
DataService:bridgeSendToServer(otherJobId, "syncScore", {
    userId = player.UserId,
    score  = DataService:get(player, "Coins"),
})

-- Route to whichever server a player is on
DataService:bridgeSendToPlayer(player.UserId, "reward", {
    item = "GoldenSword",
})
